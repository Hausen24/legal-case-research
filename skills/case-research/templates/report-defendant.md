# 报告模板：辩方场景（平台 / 被告视角）

目标：站在被告（平台）立场,回答"怎么守、怎么减损"。基于全样本客观数据,但分析重心是——哪些抗辩还有成功可能、哪些已被否定要避开、现行规则下的最优论点、降低注意义务与判赔的路径。

应用 `CLAUDE.md` 格式偏好。所有数据引用 `06_analytics.json`,案件引用带案号+法院+链接。


> **封面元信息**：报告 Markdown 开头须有一行 `**报告主题**：<简洁主题>`（用作 Word 封面主标题，简洁凝练），以及 `报告人`/`报告日期`/`检索地域`/`分析样本` 署名行（排在封面底部）。

## 报告结构

### 一、检索策略
同居中模板。

### 二、整体形势研判（对平台）
- 案件整体概况 + 六维分析,**解读落到"这对被告意味着什么"**
- 整体败诉率（**维度一**）：坦诚呈现平台在该类案件中的不利总体形势
- **维度二抗辩有效性**：这是辩方最关键的一节——把各抗辩按成功率排序,成功率高的=可主打,成功率接近 0 的=应放弃或转换思路
- **维度三判赔金额**：哪些因素推高判赔 → 我方如何抗辩以压低金额（即使败诉也减损)
- **维度四演进拐点**：规则趋严的背景下,平台还有哪些空间

### 三、各争议焦点的辩方策略
对每个争议焦点：
1. **我方仍可用的抗辩及其成功率**：该焦点下还有哪些抗辩被法院采纳过,引用支持被告的判例
2. **已被否定、应避开的抗辩**：哪些抗辩（如单纯"技术中立""算法自动推荐")已被主流判例否定,继续主张只会减损可信度
3. **现行规则下的最优论点**：即便整体不利,如何构造最有可能被采纳的抗辩组合
4. **降低注意义务/减损路径**：从胜诉或减损的被告案例中,提炼平台做了什么使法院减轻其责任（如及时全面采取措施、无获利、无应知)
5. **判赔抗辩**：针对金额计算的抗辩要点

### 四、辩方总结
- 现实形势的坦诚评估（不夸大胜算)
- 最优防御路径与减损策略（**维度六**）
- 哪些抗辩必须放弃

## 写作要求
- 立场为被告,但**必须诚实评估形势**,不能给当事人不切实际的乐观预期。
- 区分"还能打的"与"已经死的"抗辩。
- 即使大概率败诉,也要给出减损（降低判赔、减轻义务认定)的实在路径。
