# 报告模板：攻方场景（权利人 / 原告视角）

目标：站在权利人立场,回答"怎么赢"。同样基于全样本客观数据,但分析重心是——主流规则中对我方有利之处、我方应主张的论点与证据、对方抗辩的破解、可援引的有利判例、风险点。

应用 `CLAUDE.md` 格式偏好。所有数据引用 `06_analytics.json`,案件引用带案号+法院+链接。


> **封面元信息**：报告 Markdown 开头须有一行 `**报告主题**：<简洁主题>`（用作 Word 封面主标题，简洁凝练），以及 `报告人`/`报告日期`/`检索地域`/`分析样本` 署名行（排在封面底部）。

## 报告结构

### 一、检索策略
同居中模板（检索前准备、检索方案、筛选情况）。

### 二、整体形势研判（对权利人）
- 案件整体概况 + 六维分析,但**解读落到"这对原告意味着什么"**
- 整体胜诉率/支持率（**维度一**）：原告在该类案件中的总体赢面
- **维度二抗辩有效性**：对方常用抗辩的成功率——成功率低的抗辩 = 我方可放心预判其失败;成功率高的 = 我方需重点防范
- **维度三判赔金额**：影响判赔高低的因素 → 我方如何主张以争取高判赔
- **维度四演进拐点**：当前裁判趋势是否对权利人越来越有利

### 三、各争议焦点的攻方策略
对每个争议焦点：
1. **主流规则对我方有利之处**：法院在此焦点上的通常立场是否支持原告,占比多少
2. **我方应主张的论点与证据**：要赢需要证明什么（结合胜诉案例中法院认定的关键事实)
3. **对方可能的抗辩及破解**：列出该焦点下对方常见抗辩,逐一给出已被法院否定的理由（引用否定该抗辩的判例)
4. **可援引的有利判例**：2 件以上支持原告立场的典型案例（优先指导/公报案例),说明可类比之处
5. **风险点**：少数支持被告的案例提示了什么风险,如何规避

### 四、攻方总结
- 整体赢面与最优攻击路径
- 哪几个论点是制胜关键（**维度六**）
- 判赔最大化的主张要点

## 写作要求
- 立场为权利人,但**论据必须客观**——基于真实裁判数据,不歪曲。
- 明确区分"主流可依赖"与"少数需防范"。
- 给可操作的主张建议,不止描述现状。
