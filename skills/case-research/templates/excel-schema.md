# 模板：核心判决清单 Excel · 长表 Schema（v2）

> 通用基座定义，`generate_excel.py`（通用）与 `generate_excel_secmisrep.py`（证券专题）共同遵循。
> **设计原则**：废弃旧版"固定列 + 动态争点宽列"（稀疏、基本案情被挤到末列）；改为**长表**，并把"给人读"与"给机器统计"拆成不同 sheet。

## 通用样式
- 字体：表头微软雅黑加粗白字、深色底（Navy `#1F3864`）；正文宋体。
- 隔行/隔案浅灰底（`#EEF1F6`）；细边框（`#C9C9C9`）；顶端对齐、自动换行。
- 冻结首行（长表另冻结案件级首列）；开启自动筛选。
- 长文本列（基本案情、抗辩、裁判观点）列宽 ≥40；短列窄；行高足以容纳多行。
- 链接列写入超链接、显示"查看原文/查看"，hyperlink 指向裸链接。
- 零公式错误；合并单元格不破坏筛选。

## Sheet 1「判决要点」——给人读（长表）

**一案占 1 至数行**：案件级字段跨该案所有子行**合并单元格**，每个争议焦点占一**子行**，从争点开始分叉。

**案件级列（合并）**
| 列 | 要求 |
|---|---|
| 序号 | 1,2,3… |
| 案号 | 标准化案号 |
| 案件名称 | |
| 审理法院 / 审级 / 裁判日期 | |
| 裁判结果（总） | 全部支持／部分支持／驳回／撤销改判 |
| **基本案情（六要素）** | 一段完整叙事：时间·地点·主体—起因·经过·结果。让人一眼读懂；禁用"同事件群后续个案"等黑话 |
| 案件链接 | 超链接（裸链接） |

**争点级列（每个争议焦点一子行）**
| 列 | 要求 |
|---|---|
| 争议焦点 | 一个焦点一行（来自 `问题观点[*].争议焦点` / `ControversialFocus`） |
| 各方抗辩主张及理由 | 该焦点下原/被告主张与依据（来自 `问题观点[*].各方抗辩` / `DefenseViewpoint`） |
| 裁判观点及理由 | 法院对该焦点的认定与说理（来自 `问题观点[*].裁判观点`，可前置〔倾向标签〕） |

- 〔证券专题加列（案件级）〕`涉案上市公司`、`虚假陈述事件`、`核心判决类型`（示范判决/代表人诉讼/普通核心）。
- `裁判依据/法规时效核对` **不进主表**（避免臃肿），入 Sheet2 或附录。
- 实现：案件级列用 `merge_cells` 跨该案子行合并；首行写值，其余留空。

## Sheet 2「争点编码」——给机器统计（tidy，一行＝一条"案件×争点"观测）
`案号 | 审理法院 | 法院地 | 裁判年份 | 争点类别 | 裁判倾向标签 | 责任主体 | 责任比例 | 裁判结果分类 | 是否典型 | 相关度`
- 旧版稀疏的"倾向/观点"宽列全部纵向收纳于此。
- `run_analytics*` 与 `chart_theme` 出图**只读本 sheet**，保持可读表干净。

## Sheet 3「案例索引」——平行判决 + 典型案例 + 溯源
`序号 | 类别(平行判决/典型/参考) | 案件名称 | 案号 | 审理法院/入选名录 | 所属核心案 | 链接`
- 平行判决（`_track=parallel`）与典型案例（`_track=typical`）合并于此。

## 数据来源映射（生成脚本约定）
- Sheet1 案件级 ← `05_enriched_cases.json` 各核心判决的派生字段与 `基本案情（六要素）`。
- Sheet1 争点级子行 ← 该案 `问题观点` 字典逐键展开（键＝争议焦点）。一案有几个被编码的问题，就有几行子行；若某案无编码问题，占一行并在争点列写"—"。
- Sheet2 ← 同一 `问题观点` 字典展平为一行一观测。
- Sheet3 ← `04_screened_cases.json` 中 `_track∈{parallel,typical}` 的记录。
