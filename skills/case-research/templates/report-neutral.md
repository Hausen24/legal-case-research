# 报告模板：居中场景（法官 / 学术 / 中立研究）

目标：客观勾勒该类案件的裁判规则全貌,指出主流规则与分歧点,不预设立场。

应用 `CLAUDE.md` 的文风、抬头、署名等格式偏好。所有数据结论引用 `06_analytics.json`,所有案件引用带案号+法院+链接。


> **封面元信息**：报告 Markdown 开头须有一行 `**报告主题**：<简洁主题>`（用作 Word 封面主标题，简洁凝练），以及 `报告人`/`报告日期`/`检索地域`/`分析样本` 署名行（排在封面底部）。

## 报告结构

### 一、检索策略
（一）检索前准备：核心要素与争议焦点提炼
- 案件核心要素表（来自 01_elements.md）
- 核心争议焦点拆解

（二）检索方案
- 案由、三层关键词、时间范围、文书类型、审理程序、关联法条

（三）整体检索情况及成果筛选
- 检索得到 N 件 → 双轨分流 → 相关度筛查 → 保留 M 件分析样本 + P 件权威案例附录
- 筛查标准（高/中/低）

### 二、案例综述
（一）案件整体概况【六维深度分析在此展开】
- 时间分布、法院层级分布、地域分布、案件类型分布
- 整体裁判结果倾向
- **维度一交叉关联**：揭示关键变量关系（如平台类型×注意义务)
- **维度四演进拐点**：量化关键年份的标准变化
- 配合数据说明趋势,不止罗列数字

（二）争议焦点一：<焦点名>
1. 整体裁判情况：该焦点下支持各方的比例、核心理由频率（含**维度二抗辩有效性**相关数据）
2. 主流观点典型案例（2 件,支持多数立场）：每件含基本案情、该焦点具体争议、法院裁判逻辑、典型意义
3. 少数/分歧观点典型案例（2 件,支持相反立场）：同上结构
4. 小结：本焦点的主流规则 + 分歧所在（**维度五分歧地图**）

（三）争议焦点二 / （四）争议焦点三：同上结构

### 三、类案检索总结
- 综合六维,给出该类案件的裁判规则地图：哪些已成共识、哪些仍有分歧、整体趋势
- **维度六要素-结果影响度**：点明真正决定裁判的关键因素
- 中立陈述,不给某一方的策略建议

### 附录：权威案例清单
经典/评析类重要判例（案号+标题+链接）。

## 写作要求
- 学术、严谨、中立。
- 每个判断有数据或案例支撑,避免空泛。
- 正反方典型案例**对称呈现**,体现分歧而非单边。
