# 方法论：核心判决识别 · 京沪比较 · 典型案例交叉核对

> **本文件是 `group-litigation-research` 通用机制的核心**：从"一事件派生大量平行判决"中分组识别核心判决、平行判决留痕、典型案例交叉核对。下文以证券虚假陈述（京沪四家法院）为示例落地；换领域时，把"案由/目标法院/事件识别口径/典型案例名录"替换为该领域对应项，机制不变。

## 一、关键词轮替检索（绕开 MCP 限制）

`get_case_list` 只有 title/fulltext、返回前 10 条、无翻页、无法院过滤。因此：
- `title` 固定为标准案由（"证券虚假陈述责任纠纷"，必要时先核标准写法）。
- 靠 `fulltext` 不断"翻面"取不同的前 10 条，三轮轮替：
  1. **问题词轮**：揭露日 / 系统风险 / 交易因果关系 / 比例连带 / 中介机构 / 保荐 / 会计师 / 示范判决 / 代表人诉讼 / 预测性信息 / 自愿披露 …
  2. **事件/公司名轮**：检查点 1 预研得到的京沪金融法院已知重大虚假陈述案上市公司名逐一作 fulltext。
  3. **年份轮**：2018…当前，逐年作 fulltext。
- 每轮结果按 `LastInstanceCourt` 含【北京金融法院 / 上海金融法院 / 北京市高级人民法院 / 上海市高级人民法院】后过滤，并按 `DocumentAttr` 只留"判决书"；其余丢弃（保留在 03 留痕）。
- **`search_case` 语义兜底（仅发现，不直接入池）**：输入"上海/北京金融法院证券虚假陈述 示范判决 系统风险扣除"等一句话案情，它返回轻量摘要（`title/case_number/court_name/...`）。对命中的相关案件，用其 `title`/`case_number` 回查 `get_case_list` 取完整要素后才写入 03；据其结果反推新的 fulltext 关键词。
- 停止条件：三轮轮替 + 名录交叉补检后，新关键词不再带来新的核心判决；或累计 `get_case_list` 调用约 40 次。完整性以名录交叉核对兜底。

## 二、核心判决识别（去重）

证券虚假陈述的同一虚假陈述事件会派生大量平行判决（众多投资者分别起诉），只有**核心判决**有完整说理，须只保留核心判决。

步骤：
1. **Gid 去重**：去掉完全重复记录。
2. **分组**：从 `Title` 抽涉案上市公司（案件名形如"×××与/诉○○股份有限公司证券虚假陈述责任纠纷"，被告方公司即发行人）；结合 `Ascertain` 的虚假陈述事件描述（披露文件、时间段）把案件分组。**同一公司若有多次相互独立的虚假陈述事件，拆成多组。**
3. **组内选核心**，优先级：
   - 在 `Title` / `Ascertain` / `Identified` / `TrialAfter`（受理及审理情况描述）任一命中"示范判决""示范案件""代表人诉讼"（普通/特别）者 → 核心；
   - 否则取说理最完整者：字段最全（`Identified`/`RefereeBasis` 充实）、通常为二审/再审（看 `CaseClassName`/`TrialStep`）、或判赔与因果认定论述最详者；
   - `核心判决类型` 标注为 示范判决 / 代表人诉讼 / 普通核心。
4. **平行判决**：组内非核心者移入平行判决池，仅留 `CaseFlag/Title/涉案公司/Url/所属核心Gid`，不进分析、仅留痕（Excel sheet2）。
5. **典型案例池**：`CaseGrade` 非 07 的正文空记录单独入池。

> 启发式必有误差（漏标的核心判决、需拆分的多事件公司），故 **检查点 2 必须把分组与核心判定结果交给用户校核**，由律师确认后再编码。

## 三、分析主线与地域（京沪）差异处理

**主线**：编码完成后，沿 `issue-framework.md` 的构成要件五维一条线（责任主体→虚假陈述行为→损害结果→因果关系→主观过错＋程序事项），对每个问题统计全样本"倾向标签"整体分布（`run_analytics_secmisrep.py` 的 `各问题倾向整体分布`）。报告第二章即按五维 × 四段（争点界定 / 主流规则含代表案 / 例外与反向 / 评析要点）深度展开，整体裁判数据为骨、全文说理为肉。

**地域差异不是组织轴线，由样本量闸门裁定**：是否输出"京沪/地域分歧"结论，由 `scripts/common/stats_guard.py` 的 `divergence_gate({地: 件数})` 依样本量决定（任一地 < 阈值即判 `report=False`）。
- `report=False`（小样本，常态）：**不单设任何地域分歧章节或逐问题小节**；至多在结论处依其 `phrasing` 一句话交代"两地样本悬殊、不作地域统计推断"；如确有个案取向不同，仅在相关争点处一句话点出。
- `report=True`（两地样本均达阈值）：方可就确有差异的问题，引数据指出分歧点（取向或比例尺度不同），成因解读结合制度背景但不臆断，以数据与判决文本为据。
- 任何情况下都不得为凑差异而编造；无实质分歧如实写"本批无明显地域差异"。

## 四、典型案例名录交叉核对（需联网）

- 联网检索四家法院历年公布的"典型案例 / 十大案例 / 参考性案例 / 年度审判白皮书典型案例"中涉证券虚假陈述者（如"上海金融法院 典型案例 证券虚假陈述""北京金融法院 典型案例"）。
- 交叉：
  - 名录案件在样本中找到 → `是否典型案例` 标注名录全称；
  - 名录有、样本未检索到 → 列入缺口，回检索步用该案上市公司名补查 `get_case_list`（或先用 `search_case` 发现再回查）；仍检不到则在报告标"该典型案例未在 MCP 库内检索到，仅据公开名录列示"。
- **红线**：名录与评介可联网获取，但任何裁判事实/裁判观点/案号引用仍以 MCP 返回为准，不得以网络转述替代库内数据、不得编造。
