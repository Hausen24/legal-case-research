## v1.1.2 —— 「」豁免通道哨兵：不核验 ≠ 不可见
- **哨兵 WARN**：「」按纪律只装短术语、不进引文核验——把编造"判决原话"包进「」即可绕过
  核验层（prompt 级纪律的固有缺口）。verify_report 新增哨兵：≥20 字、或含裁判语体特征
  （本院认为/应当/构成等）的「」片段提示人工抽核；命中语料者自动豁免。**不阻断**（即使
  `--strict-quotes`），只把豁免通道置于可见状态。
- **边界声明补窄**：README/docstring 同步写明「」标记的归纳语亦不在核验范围——宁可宣称窄、
  不可宣称宽。模板 §2c 增反向禁令：不得把判决原话包进「」逃避核验。
- 哨兵首扫即抓到 demo 中 5 条超长「」归纳语，已去引号归位；新增 2 项测试
  （哨兵正反用例 + demo 无警示回归），41 项全绿。

## v1.1.1 —— 引文层补丁：旗标过滤 + 引号写作纪律 + CI strict 回归
- **修复**：verify_report 显式传报告文件名时 `--strict-quotes` 被当文件名导致必然 FAIL——
  构建报告清单前过滤 `--` 开头参数；新增回归测试。
- **引号写作纪律**（模板 §2c 固化）：弯引号“”专用于逐字引用（受核验），直角引号「」用于
  术语/归纳语（不核验）；核验正则相应只认弯引号。三套 demo 报告引号全部归位。
- **CI strict 回归保护**：两条冒烟管道的 verify 升级为 `--strict-quotes`，引文层从此有回归防线。
- **模糊≠逐字边界注明**：脚本 docstring 与 README 写明「模糊命中≈2/3 以上连续片段来自
  原文，不保证逐字一致」；未命中提示措辞同步校准。

## v1.1 —— 外部评审回应：引文核验层 + 编码抽检 + 诚实边界声明
针对外部评审五点批评逐一落实：
- **引文核验层（内容级抽检）**：verify_report 新增第二层——报告中引号内直接引文在判决
  全文语料（全池正文字段+output/原文/）归一化匹配（嵌套引号容错、分块模糊判定），
  逐字引用判决原话应当命中、**伪造"判决原话"会被标出**；未命中清单提示人工抽核，
  `--strict-quotes` 升级 FAIL。实测：真实原话精确命中、伪造引文未命中。
- **措辞收敛（诚实边界）**：README/SKILL/AGENTS 全面把"反幻觉被机器证明"收敛为
  "引用与引文可溯源性被机器证明"，明示**不能证明全部转述内容无误**，转述质量由编码
  抽检与人工检查点把关；英文 tagline 改 machine-verified citation traceability。
- **编码抽检机制**：spot_check_coding.py 随机抽 ≥15%（至少 3 件）生成"编码结论 vs
  原文摘录"对照复核单，随检查点交用户逐件确认留痕——内容分析编码者信度的工程化替代
  （双跑 Kappa 入 ROADMAP）；SKILL 设为强制工序。
- **编码落盘标准化**：apply_coding.py——AI 产纯 JSON 编码合并入 05，受保护 MCP 原始
  字段拒绝改写（字段保全机器强制），收编"每次研究手写脚本"的手工环节。
- **认知负荷与兼容性**：SKILL 顶部新增"铁律速查卡"（10 条×违反后果）；AGENTS 兼容性
  表述收敛为"架构设计目标，判断层仅在 Claude Opus 级完整验证"；跨模型 eval 入 ROADMAP。
- **召回率天花板明示**：README 写明"量化不等于消除——单查询 10 条上限是结构性约束"。
- 新增 2 项测试（37 项全绿）。

## v0.7 —— 双模式：实案研究 / 学理研究（二期）
- **双研究模式**：case-research 技能开工时选择——学理研究（全国检索）或实案研究（指定核心法院，按法发〔2020〕24号第四条四顺位检索：指导性案例→最高法→本省高院→上一级及本院）。
- **指导性案例本地数据集** `data/guiding_cases/`（263 案全文）：`fetch_guiding_cases.py` 自最高法官网合规限速抓取（自动翻页 15 页、断点续传、失败重试），含关键词/裁判要点/基本案情/裁判结果/裁判理由/相关法条结构化解析；官网未收录的 16 个编号（疑似废止下架）如实记录于数据集 README，引用前按第九条核查效力。
- **四顺位法院解析** `court_hierarchy.py` + `data/court_hierarchy.json`：直辖市多中院映射（朝阳→北京三中院等）、专门法院上诉关系、非直辖市通用规则推断；提议制——检查点 1 必须用户确认后落盘 `顺位法院.json`。
- **相关度 A/B/C 方法论**：A 套高贴合（高相关候选，≈90%+ 标识）/B 套概念放大（中相关候选，≈80%+ 标识）/C 套兜底（仅不足时启用+特别提示）；终档按第六条三要素比对量表（基本事实/争议焦点/法律适用），每案记`相关度依据`；百分比明示为分级标识非计量值。
- **报告新结构**：「案情要素拆解与检索映射表」小节（要素×案情拆解×A/B 套词）；学理模式地域分布图 `region_dist` + 争点×地域观点热力矩阵 `issue_region`（描述性，受闸门约束）且不设控辩表；实案模式新增「类案规则（依据《类案检索规定》）」专章（逐顺位三要素比对+裁判逻辑拆解+应当参照/可以参考效力标注）、附录满足第八条要素。
- `check_coverage.py` 顺位覆盖节落地（读 `顺位法院.json` 逐顺位统计命中与缺口）。
- 新增 6 项二期测试（33 项全绿）。
