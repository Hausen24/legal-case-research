# 更新日志

本项目版本号遵循「功能里程碑」粒度，非严格 SemVer。

## v0.6 —— 质量门三件套（数据层与质量层，双模式一期）
- **检索覆盖率自检** `scripts/check_coverage.py` → `07_coverage.json`：关键词命中矩阵（含"独有命中"边际贡献）、案件等级/法院层级分布、去重审计、典型案例名录命中/缺口表（`名录.json`）——把"完整性不可保证"变成量化缺口表，亦是法发〔2020〕24号第八条"方法/结果"要素的底稿；为实案模式预留"顺位覆盖"节。
- **管道数据契约校验** `scripts/common/pipeline_schema.py` + `scripts/validate_pipeline.py`：03–06 逐段校验字段类型/枚举/焦点名拼写一致性（拼写漂移会导致聚合分裂），宽松白名单设计（新字段不报错，二期零返工）；两套 run_analytics 内置 warn 模式复查。
- **分歧检测一等产出（数据层）** `scripts/common/divergence.py`：06 新增 `分歧地图`（同争点对立立场并存清单 + 各立场代表案对 + 地域观察 + 闸门措辞，法发〔2020〕24号第十一条情形显性化）；两版 Excel 新增「裁判分歧清单」sheet。
- **ROADMAP.md**：固化二期"双模式（实案研究/学理研究）"完整施工图（四顺位检索引擎、相关度 A/B/C 方法论、案情要素拆解映射表、地域观点矩阵、第八/九条合规等）。
- 《类案检索指导意见》文号经法宝法规 MCP 核验：法发〔2020〕24号，现行有效。
- 新增 10 项质量门测试（27 项全绿）；CI 两个 demo 冒烟接入 validate + coverage。

## v0.5 —— 工作流再抽象 + 通用化 + 命名规范
- **工作流分类重构**：证券虚假陈述专题（`securities-misrep-research`）抽象为通用 **`group-litigation-research`（集团/群体性诉讼）** 工作流——核心区别是"从一事件派生的大量平行判决中按主体+事件分组、筛出核心实质判决"；证券虚假陈述降为内置示范样例。
- **固定报告/清单命名规范**：`<案件类别>-类案检索报告/清单-<YYYYMMDD>`。
- **通用性**：自带数据模式（`import_cases.py` + 输入数据契约，无需法宝订阅）；`AGENTS.md` 工具无关运行指南（不限 Claude）。
- **内外分离**：个人画像 `CLAUDE.md` → `CLAUDE.example.md` 模板 + gitignore。
- **展示样板**升级为证券虚假陈述集团诉讼合成 demo（README hero）。

## v0.4 —— 开源加固（可展示 / 可复现 / 可证明）
- **新增 LICENSE（MIT）**，使项目可被合法使用与二次开发。
- **新增脱敏演示样本** `examples/`：合成一份与 MCP 返回结构一致的小样本（虚构案号/当事人，见 `DISCLAIMER.md`），**无需法宝 Token 即可跑通整条管道**并产出报告/Excel/图表，作为 GitHub 展示与测试夹具。
- **新增反幻觉收尾自检** `scripts/verify_report.py`：扫描成品报告每一个案号，逐一比对样本池，引用了样本池外的案号即报错——把"承诺不编案号"升级为"机器证明可溯源"。已接入两套技能的收尾流程与 CI。
- **新增自动化测试与 CI**：`tests/` 下 pytest 覆盖公共派生、样本量自适应闸门（含"小样本不作分歧推断"回归保护）、Excel 实质列非空、反幻觉校验正反用例；GitHub Actions 每次推送/PR 运行测试 + 演示管道冒烟 + 反幻觉校验。
- README 增补徽章、英文简介、Demo 展示区（图表内联）、可靠性章节。
