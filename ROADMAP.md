## 二期：双模式（实案研究 / 学理研究）

### 法律依据（已核验）

《最高人民法院关于统一法律适用加强类案检索的指导意见（试行）》，**法发〔2020〕24号**，
2020-07-31 试行，现行有效（北大法宝 Gid `0749b01d6f2da00dbdfb`）。关键条款：

- **第四条（检索范围四顺位）**：①最高法指导性案例 → ②最高法典型案例及裁判生效案件 →
  ③本省（自治区、直辖市）高院参考性案例及裁判生效案件 → ④上一级法院及本院裁判生效案件。
  除指导性案例外**优先检索近三年**；前一顺位检索到类案的，可以不再检索。
- **第六条（相似性识别与比对）**：要求承办法官"将待决案件与检索结果进行相似性识别和比对"——
  相关度量表是这一比对动作的量化落地。三要素（基本事实、争议焦点、法律适用问题）本身出自**第一条**
  类案定义（原文"…等方面具有相似性"，"等"表非穷尽）；勿把定义条与比对条混引。
- **第八条（检索说明/报告要素）**：检索主体、时间、平台、方法、结果、类案裁判要点、
  待决案件争议焦点 + 是否参照/参考的分析说明——**我们报告的合规验收清单**。
- **第九条（效力分层）**：指导性案例**应当参照**（与新法/新指导性案例冲突的除外）；
  其他类案**可以参考**——实案模式列案时的两档效力标注。
- **第十一条（分歧解决）**：检索到类案法律适用不一致 → 分歧解决机制——分歧地图的官方注脚。

### A. 模式选择（intake）

SKILL 第一步新增"研究模式"询问：
- **学理研究**（默认）：全国范围检索、不限法院，产出裁判规则研究。
- **实案研究**：用户**必须指定核心法院**（如"北京市朝阳区人民法院"），按法发〔2020〕24号
  四顺位检索，产出可随案归档的类案检索报告。
- 时间范围以用户输入为准；实案模式提示"规定建议优先近三年（指导性案例除外）"作默认推荐。

### B. 相关度方法论（两模式通用，A/B 两套关键词）

- **A 套关键词（高贴合）**：尽可能贴合用户案情 → 高相关候选池。
- **B 套关键词（概念放大/偏移）**：上位概念、相邻行为 → 中相关候选池。
- **C 套（兜底扩展）**：仅当 A+B 检索结果不足时启用，结果一律标低相关 + **报告特别提示**。
- **候选池 ≠ 终判**：逐案筛查按**第六条三要素比对量表**定档——
  高（≈90%+ 标识）＝基本事实与争议焦点均高度吻合（仅当事人/数额/地点等非实质差异）；
  中（≈80%+ 标识）＝争议焦点与法律适用问题相同，基本事实存在场景偏移；
  低＝仅法律适用问题相关。百分比为**分级标识**而非计量值（脚注注明）。
  每案记 `相关度` + `相关度依据`（一句话）。
- 落点：`methodology/keyword-strategy.md`（A/B/C 套）、`methodology/relevance-screening.md`
  （三要素量表）、报告"案情要素拆解与检索映射"表（见 C）。

### E. 实案模式（新）

1. **顺位法院解析**：核心法院 → 上一级法院 → 本省高院 → 最高法。
   - `data/court_hierarchy.yaml`：直辖市"基层→中院"映射数据（北京一/二/三/四中院、
     上海一/二中院、天津、重庆各中院辖区），其余地区 AI 提议。
   - **检查点 1 必须呈报顺位法院清单供用户确认**（安全网）。
2. **顺位检索引擎**：复用"关键词轮替 + `LastInstanceCourt` 后过滤"机制，按四顺位分档；
   默认**全顺位检索**（报告更全面），标注规定的"前顺位命中可止步"规则与首个命中顺位。
   `check_coverage.py` 填充 `顺位覆盖` 节（每顺位：目标法院/命中数/案件清单/缺口）。
3. **新章节**："二、类案规则（依据《类案检索规定》）"插在"裁判规则全景"之前。
   逐顺位列高/中相关案件，每案深度比对：
   - 相似性比对（第六条三要素：基本事实/争议焦点/法律适用 逐项对照本案）；
   - 逐争议焦点拆解该院裁判逻辑（请求权基础→抗辩→说理→结论）；
   - **效力标注**：指导性案例标"应当参照"（含与新法冲突例外核查），其他标"可以参考"（第九条）。
4. **指导性案例全文**（顺位①）：已拍板两步走——
   - 第一步：MCP `CaseGrade` 识别 + 元数据/链接列示，深度比对留〔全文补实〕占位或人工补；
   - 第二步：合规整理最高法官网已公布指导性案例为 `data/` 静态数据集 + 更新脚本
     （官方文书不受著作权法保护；数量有限、更新慢，静态数据集最稳）。
5. **第八条合规**：报告附录"检索过程说明"逐项覆盖第八条要素（主体/时间/平台/方法/结果/
   裁判要点/争议焦点/参照参考分析），数据源 = `07_coverage.json` + 顺位覆盖。
6. 直辖市/自治区多中院提醒（如朝阳→北京三中院）写入 SKILL 关键事实。
